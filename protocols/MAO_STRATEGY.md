## 2. Core Algorithms / 核心算法

### 2.1 PCDR (Principal Contradiction Dimensionality Reduction)
### 2.1 主要矛盾降维算法 (PCDR)
- **Logic**: In a multi-constraint environment, the system calculates the gradient of each variable relative to the global objective function. It then extracts the "Principal Eigenvector" as the main target.
- **逻辑**: 在多约束环境下通过计算各变量对全局目标函数的梯度，提取特征值最大的“主轴”作为主攻方向。
- **Action**: Reject noise; allocate 80% of compute to the PCDR node.

### 2.2 ALPA (Asymmetric Local Pressure Algorithm)
### 2.2 局部非对称压强算法 (ALPA)
- **Logic**: Concentration of force. Once a target node is identified, the system reallocates resources to achieve at least a **3:1** ratio against the constraint, ensuring a "total annihilation" stroke.
- **逻辑**: 集中兵力。一旦锁定目标节点，执行 3:1 以上的局部资源重置，追求“速决”与“全歼”。

### 2.3 CAFP (Contradiction Aspect Flipping Prediction)
### 2.3 矛盾地位反转预测 (CAFP)
- **Logic**: Determines the subject-object relationship. Calculates the critical tipping point ($\Delta E$) where the balance of power shifts from defense to offense.
- **逻辑**: 判定主客体地位，计算势能反转临界点 ($\Delta E$)，识别从战略防御转向战略反攻的时机。

### 2.4 SAA (Sense-Assess-Act / Sparrow Dissection)
### 2.4 “解剖麻雀”感知闭环 (SAA)
- **Logic**: Rejects aggregated reporting. Triggers a "Sparrow Dissection" protocol to verify environment truth via raw byte-stream/log analysis.
- **逻辑**: 拒绝汇总报告。触发“解剖麻雀”协议，通过穿透层级的原始数据分析（Deep-dive）校验环境真相。

## 3. Negative Constraints (Dark Side Protocols)
## 3. 底线红线 (暗面协议)

### 3.1 Anti-Adventurism / 防范冒进
- **Rule**: IF `Cost > Recovery_Rate` AND `Local_Pressure < 3`, the operation is VETOED. Avoid "Defending the Gate" in favor of strategic retreat.
- **规则**: 若总耗散大于恢复率且局部压强不足 3 倍，系统强制一票否决。严禁“御敌于国门之外”的冒险主义。

### 3.2 Mass-Line Aggregation / 群众路线众包
- **Rule**: IF `Confidence < Threshold`, downgrade central authority. Broadcast sub-tasks to the edge agent network for weighted ensemble.
- **规则**: 若置信度低于阈值，降级中心权限，广播任务至边缘 Agent 网络进行加权合并。
